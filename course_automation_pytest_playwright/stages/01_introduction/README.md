# 01 — Введение

Цели этого этапа

- Понять, что такое автоматизация тестирования
- Ознакомиться с инструментами курса: Python, pytest, Playwright, GitLab CI, Docker
- Установить и подготовить окружение (следующий этап детализирует это)

Теория (кратко)

- Автоматизация тестирования позволяет запускать проверки повторяемо и быстро.
- Unit tests проверяют отдельные функции/модули.
- Integration tests проверяют взаимодействие между компонентами.
- E2E (end-to-end) тесты симулируют поведение пользователя (UI) и чаще всего дороже в поддержке.

Практическое задание — прочитать и ответить

1. Установите Python 3.10+ и создайте виртуальное окружение.
2. Прочитайте официальные docs: pytest и Playwright (ссылки есть в корневом README).
3. Запишите в `answers.md` (в той же папке) короткие ответы: зачем нужен unit тест, integration, e2e.
